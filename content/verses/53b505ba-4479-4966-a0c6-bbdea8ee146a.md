本文真正的 Title 是“我为什么要关心 Kubernetes, Docker 和容器编排？”。理由是它让我们 once for all 的做了以下这些事情：

* OS 版本，编程语言的版本及其依赖。
* 代码打包，代码的依赖，以及可能的补丁。
* 简单地运维，扩容（docker run / kubectl apply xyz.yml)
* Dockerfile 声明了应用所需要的东西，K8S YAML 生命了端口，replica 数量，名字，环境变量等等。
